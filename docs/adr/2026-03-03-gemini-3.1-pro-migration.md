# ADR: Migrate gemini-3-pro-preview to gemini-3.1-pro-preview

## Status

Accepted — 2026-03-03

## Context

Google is deprecating `gemini-3-pro-preview` with removal scheduled for the end of March 2026. This change was necessary to maintain compatibility with Google's API roadmap. The project previously referenced the deprecated model in source code, tests, and documentation.

## Decision

Replace all references to `gemini-3-pro-preview` with `gemini-3.1-pro-preview` across the codebase and update the @google/genai SDK to the latest stable version. A new configuration migration helper script was created to assist users in updating their environment variables and MCP client configuration files.

### Model Name Mapping

| Old | New |
|-----|-----|
| gemini-3-pro-preview | gemini-3.1-pro-preview |
| gemini-3-pro-image-preview | gemini-3.1-pro-image-preview |

### SDK Update

Updated @google/genai from ^1.42.0 to ^latest via `npm install @google/genai@latest`. The package.json uses caret range (e.g., ^1.43.0) per project convention, with exact version pinned in package-lock.json.

### Regex Compatibility

The existing regex `/gemini[-_]?3/` in GeminiAIService.ts already matches both old and new model names because both contain the substring `gemini-3`. No regex changes were required.

### Default Models Unchanged

- Default query model remains `gemini-3-flash-preview` (not affected by deprecation)
- Default image model remains `gemini-2.5-flash-image` (not affected by deprecation)

### Configuration Migration Helper Script

Created `scripts/migrate-model-config.sh` as a configuration migration helper that:

1. Checks `GEMINI_MODEL` environment variable for deprecated model names
2. Scans common MCP client config locations for deprecated model strings:
   - `~/.claude/claude_desktop_config.json`
   - `~/.cursor/mcp.json`
   - `~/.config/gemini-mcp-server/.env`
3. Outputs specific replacement commands or sed one-liners
4. Returns exit code 0 if no deprecated models found, 1 if deprecated models detected

**Note**: This project is a stateless MCP server with no persistence layer. The "database migration" request was reinterpreted as a configuration migration helper, as the only form of persistence is user environment variables and MCP client configuration files.
